# PatentRank — 특허 가치 평가 5축 스코어 공식

---

## 0. 한눈에 보기

**무엇을 하는가**
국내 등록 특허 한 건을 입력받아 **5개 축의 점수를 계산하고, 동일 기술 분야 내에서 비교한 뒤 AAA부터 C까지 9등급을 부여**한다.

**5개 축**

| 축 | 무엇을 보는가 | 가중치 |
|---|---|---:|
| **INV** 혁신성 | 청구항 범위와 기술 다양성 | 0.25 |
| **IMP** 영향력 | 다른 특허들이 얼마나 인용했는가 | 0.20 |
| **MKT** 시장성 | 어떤 산업에 얼마나 큰 시장이 있는가 | 0.20 |
| **NET** 네트워크 중심성 | 인용 그래프에서의 위치 | 0.20 |
| **COM** 사업화 준비도 | 실제로 사업화될 가능성 | 0.15 |

**전체 흐름**

```
원시 데이터 (KIPRIS, NTIS, KOSIS 등)
     ↓
변수별 변환 (로그, 엔트로피, Gini 등)
     ↓
동일 기술군 내 백분위 정규화 → 모든 변수가 [0, 1]
     ↓
5축 가중합 → 원점수 (0 ~ 1)
     ↓
WIPO 기술군 내 스테나인 9등급 분할
     ↓
AAA ~ C 등급 부여
```

## 2. 변수 정규화 — 모든 변수를 [0, 1]로

### 2.1 왜 정규화가 필요한가

원시값은 단위와 분포가 제각각이다. 청구항은 보통 5~20개, 패밀리는 1~50국, 피인용은 0~수천 건. 게다가 같은 "청구항 10개"가 화학에서는 평범하지만 IT에서는 큰 값일 수 있다. **모든 변수를 같은 척도([0, 1])에 올리고, 동일 분야 안에서 비교**해야 공정한 점수가 나온다.

### 2.2 처리 단계

```
원시값(raw) → 변환(transform) → 같은 분야끼리 비교(normalize)
```

| 변수 유형 | 예시 | 변환 | 비교 방식 | 출력 |
|---|---|---|---|---|
| 카운트형 | 청구항 수, 피인용 수, 패밀리 수 | `log(1+x)` | 코호트 내 백분위 | [0, 1] |
| 다양성 | IPC 분류 분포 | Shannon 엔트로피 ÷ 최대값 | 변환만으로 충분 | [0, 1] |
| 금액·규모 | TAM, 과제 예산 | `log(1+x)` | 코호트 내 백분위 | [0, 1] |
| 비율 | CAGR | (값 그대로) | 코호트 내 백분위 | [0, 1] |
| 네트워크 지표 | PageRank, Betweenness | `log(1+x)` | 코호트 내 백분위 | [0, 1] |
| 시계열 분산 | TemporalPersistence | `1 − Gini` | 코호트 내 백분위 | [0, 1] |
| 이진 | IsSEP, HasSpinoff | 변환 없음 | 그대로 | {0, 1} |

### 2.3 핵심 수식

**(a) 백분위 순위**

코호트 안에서 이 특허가 몇 % 위치인지 계산한다.

```text
PR(x | C) = (rank(x; C) − 0.5) / |C|
```

- 코호트 `C` 안에서 `x`의 오름차순 순위
- `−0.5`는 양 끝점(0과 1)에 너무 가까워지지 않도록 보정
- 결과는 항상 `[0, 1]`

**예시**: 100개 특허 중 80등이면 → (80 − 0.5) / 100 = 0.795

> **왜 z-score가 아닌 백분위인가**
> 인용 수처럼 한쪽으로 극단적으로 치우친 분포(소수 특허가 인용 대부분을 가져감)에서는 z-score가 극단값에 끌려간다. 백분위는 분포 모양에 무관하게 안정적이다.

**(b) 카운트형 변수 정규화**

먼저 로그를 씌워 분포를 완만화한 뒤 백분위.

```text
NormCount(x | C) = PR( log(1 + x) | C )
```

> 인용 0건과 1건의 차이는 100건과 101건의 차이보다 의미가 크다. 로그 변환이 그 직관을 반영한다.

**(c) 엔트로피 (다양성) 변수**

```text
H(x) = −Σ pᵢ · log(pᵢ)
NormDiv(x) = H(x) / log(N_obs)
```

- `pᵢ`는 각 카테고리(예: IPC 서브클래스)의 비율
- `N_obs`는 관측된 카테고리 수
- 출력이 자연스럽게 [0, 1] → **코호트 비교 불필요**

**(d) 시계열 분산 (TemporalPersistence)**

```text
TP(p) = 1 − Gini( c_1(p), c_2(p), …, c_N(p) )
```

- `c_y(p)`: 등록 후 y년차 피인용 수
- Gini 계수: 0 = 균등, 1 = 한 해에 집중
- `1 − Gini` = 꾸준한 정도

> 같은 누적 인용 100건이라도 "초기 3년에 100건 후 0건" vs "20년에 걸쳐 매년 5건"은 완전히 다른 신호.

### 2.4 코호트 — 누구와 비교할 것인가

백분위 정규화의 핵심은 **누구와 비교하느냐**다.

| 변수군 | 코호트 | 비교 시점 |
|---|---|---|
| INV (혁신성) | 같은 WIPO 분류 + 출원연도 ±2년 | 출원 시점 신호이므로 |
| IMP, NET (인용 기반) | 같은 WIPO 분류 + 등록연도 ±2년 | 인용은 시간 누적이므로 연차 기준 필수 |
| MKT — FamilySize | 같은 WIPO 분류 + 출원연도 ±2년 | 패밀리 확장은 분야별 비용 다름 |
| MKT — IndustryAlignment | KSIC 산업코드 단위 | KSIC-IPC 연계표 사용 |
| COM | 같은 WIPO 분류 (시점 무관) | 사업화 신호는 시간 누적 |

**WIPO 4대 기술분류**

| 분류 | 영역 |
|---|---|
| 전기·전자·IT | 통신, 반도체, 소프트웨어, 디스플레이 |
| 기계 | 기계요소, 동력기관, 공작기계, 운송기계 |
| 기구 | 측정·제어 기기, 의료기기, 광학기기 |
| 화학 | 유기·무기 화학, 고분자, 의약, 식품, 환경 |

**코호트 구성 로직**

```text
def cohort(특허 p, 축 a):
    분야 = WIPO_분류(p의 주 IPC)
    기준연도 = p의 출원연도 (INV의 경우)
              or p의 등록연도 (IMP, NET, FamilySize의 경우)
              or 없음 (COM의 경우)

    if 기준연도 있음:
        C = {분야 같고, 연도가 기준연도 ±2년 안에 있는 모든 특허}
    else:
        C = {분야가 같은 모든 특허}

    if |C| < 30:
        # 통계적 안정성을 위해 코호트 확장
        # ±2년 → ±5년 → 무제한 → WIPO 분류 무시
        확장하여 |C| ≥ 30 만족시키기
    return C
```

> **|C| ≥ 30**: 통계적 안정성을 위한 최소 코호트 크기. 이보다 작으면 백분위가 신뢰 불가.

## 3. 데이터 모델 — 그래프 구조

5축 변수 일부는 특허 한 건의 속성만으로 계산되지만, **NET 축과 일부 IMP/COM 변수는 그래프 전역 연산**이 필요하다. 두 종류의 그래프를 명시적으로 사용한다.

### 3.1 특허 인용 그래프 (PCG)

특허 사이의 인용 관계를 방향성 그래프로 모델링한다.

```text
G_PCG = (노드, 엣지)
  노드: 평가 대상 특허들
       속성 = {특허번호, 출원연도, 등록연도, 주 IPC, 전체 IPC,
              출원인, 발명자, WIPO 분류, 패밀리 ID}
  엣지: citing → cited (인용자 → 피인용자)
       속성 = {인용 유형(심사관/출원인/자가), 인용 일자}
```

- **자가 인용**: 같은 출원인의 자기 인용은 가중 0.5로 감쇠하거나 제외 가능
- **시간 절단**: 코호트 정규화를 위해 등록연도 기준으로 부분 그래프 추출 가능해야 함

### 3.2 발명자 협력 그래프 (ICG)

발명자 간 공동발명 관계를 무방향 가중 그래프로 모델링한다.

```text
G_ICG = (노드, 엣지)
  노드: 발명자들
       속성 = {발명자 ID, 소속기관 변경 이력, 누적 특허 수}
  엣지: 공동발명 관계
       속성 = {가중치 = 공동발명 건수, 첫/마지막 협력 일자}
```

- **발명자 동명이인 처리**: `(이름 + 소속기관 + IPC 섹션)` 조합으로 1차 식별, 특허고객번호로 2차 확정

### 3.3 변수별 데이터 의존성

| 변수 | PCG 필요 | ICG 필요 | 계산 방식 |
|---|:---:|:---:|---|
| INV — ClaimBreadth | — | — | 특허 속성 |
| INV — IPCDiversity | — | — | 특허 속성 |
| INV — BackwardCitationDiversity | ◯ 엣지만 | — | 인용한 특허들의 IPC 분포 |
| IMP — ForwardCitationCount | ◯ 엣지만 | — | 인입 엣지 카운트 |
| **IMP — ForwardCitationQuality** | ● 전역 | — | PageRank 평균. 전역 PageRank 선계산 필요 |
| IMP — CitationSpan | ◯ 엣지만 | — | 인용자 특허들의 IPC 분포 |
| MKT — 모든 변수 | — | — | 특허 속성 + 외부 매핑 |
| **NET — PageRank** | ● 전역 | — | 전역 그래프 연산 |
| **NET — Betweenness** | ● 전역 | — | 전역 그래프 연산 (필요 시 sampling 근사) |
| NET — CrossDomainBridgeScore | ◯ 엣지만 | — | 인용/피인용 IPC 쌍 비교 |
| NET — TemporalPersistence | ◯ 엣지만 | — | 인입 엣지 연도별 분포 |
| COM — 대부분 | — | — | 외부 DB 매칭 |
| **COM — CollaborationCentrality** | — | ● 전역 | ICG에서 연결중심성 |

> 범례: `●` 전역 그래프 연산 필요 / `◯` 엣지 목록만 있으면 됨 / `—` 그래프 불필요

### 3.4 계산 순서

```text
1단계 — 전역 선계산:
   PCG 구축 → 모든 노드의 PageRank 계산
   ICG 구축 → 모든 발명자의 연결중심성 계산

2단계 — 특허별 변수 산출:
   INV, IMP, NET, MKT, COM 각 변수를 위 결과 활용해 계산

3단계 — 코호트 정규화:
   §2.4의 코호트 기준으로 변수별 백분위 적용

4단계 — 5축 가중합 → 스테나인 → 등급:
   §9의 절차에 따름
```

### 3.5 권장 인프라

| 항목 | 권장 |
|---|---|
| 데이터베이스 | PostgreSQL (원천 데이터, 인용 테이블) |
| 그래프 라이브러리 | igraph 또는 graph-tool (C 구현, 수십만 노드 가능) |
| Betweenness | 노드 30만 초과 시 Brandes sampling 근사 |
| 갱신 주기 | PCG 월 1회, ICG 분기 1회 (그 후 코호트 재구성 → 점수 재산출) |

## 4. 축 1 — INV 혁신성

```text
INV(p) = 0.4 × ClaimBreadth
       + 0.3 × IPCDiversity
       + 0.3 × BackwardCitationDiversity
```

청구항은 출원 시점에 즉시 확정되는 강한 신호이므로 0.4. 다양성 지표는 분류 체계와 심사관 재량에 영향받는 약한 신호이므로 각 0.3.

### 4.1 ClaimBreadth (가중 0.4)

**한 줄 정의**: 권리 범위가 얼마나 넓은가

| 항목 | 내용 |
|---|---|
| 원시값 | 독립청구항 수 |
| 변환 | `log(1 + 독립청구항 수)` |
| 정규화 | 같은 WIPO 분류 + 출원연도 ±2년 코호트 내 백분위 |
| 출력 | [0, 1] |
| 데이터 | KIPRIS Plus 명세서 |

**왜 중요한가**: 독립청구항이 많을수록 권리 범위가 넓고, 회피 설계가 어려워진다. 분야별 평균이 다르므로(화학 vs IT) 동일 분야 내 비교 필수.

**참고 문헌**: Tong & Frame (1994), Lanjouw & Schankerman (2001)

### 4.2 IPCDiversity (가중 0.3)

**한 줄 정의**: 한 특허가 얼마나 여러 기술 분야에 걸치는가

| 항목 | 내용 |
|---|---|
| 원시값 | 본 특허에 부여된 IPC 서브클래스 분포 |
| 변환 | Shannon 엔트로피 ÷ log(관측 카테고리 수) |
| 정규화 | 변환만으로 [0, 1] 보장 → 코호트 비교 불필요 |
| 출력 | [0, 1] |
| 데이터 | KIPRIS Plus 서지정보 |

**왜 중요한가**: 여러 IPC에 걸치는 특허는 융합기술 또는 플랫폼 기술인 경향. 엔트로피 자체가 0~최대값으로 자연 정규화되므로 분야 보정 없이 사용 가능.

**참고 문헌**: Trajtenberg, Henderson & Jaffe (1997) originality 지표, OECD 2013

### 4.3 BackwardCitationDiversity (가중 0.3)

**한 줄 정의**: 어디서 영감을 받았는가 — 선행 특허의 다양성

| 항목 | 내용 |
|---|---|
| 원시값 | 인용한 선행 특허들의 IPC 분포 |
| 변환 | Shannon 엔트로피 ÷ log(관측 카테고리 수) |
| 정규화 | 변환만으로 [0, 1] |
| 출력 | [0, 1] |
| 데이터 | KIPRIS Plus 인용정보 |

**왜 중요한가**: 다양한 분야에서 지식을 끌어왔다는 것은 새로운 결합 = 슘페터적 혁신의 신호. 인용이 없는 특허(약 15%)는 결측 처리하고 IPCDiversity 가중치를 +0.1 만큼 옮긴다.

**참고 문헌**: Carpenter, Narin & Woolf (1981), OECD 2013

## 5. 축 2 — IMP 영향력

```text
IMP(p) = 0.5 × ForwardCitationCount
       + 0.3 × ForwardCitationQuality
       + 0.2 × CitationSpan
```

단순 피인용 수가 가장 강한 신호이므로 0.5, 인용 품질로 보완(0.3), 분야 간 인용은 노이즈가 있어 보조 신호로 0.2.

### 5.1 ForwardCitationCount (가중 0.5)

**한 줄 정의**: 다른 특허들이 얼마나 많이 인용했는가

| 항목 | 내용 |
|---|---|
| 원시값 | 등록 후 받은 피인용 건수. 심사관 인용 가중 1.5, 출원인 인용 가중 0.7 |
| 변환 | `log(1 + 가중 합계)` |
| 정규화 | 같은 WIPO 분류 + 등록연도 ±2년 코호트 내 백분위 |
| 출력 | [0, 1] |
| 데이터 | KIPRIS Plus 피인용정보 |

**왜 중요한가**: 5축 전체에서 단일 변수로 가장 큰 가중치(0.20 × 0.5 = 0.10). 학술 합의가 가장 두터운 가치 신호. 등록연도 기준 코호트 정규화로 "오래된 특허만 유리해지는" 문제를 자동 해소.

**참고 문헌**: Trajtenberg (1990), Harhoff et al. (1999), Hall, Jaffe & Trajtenberg (2005), Hegde & Sampat (2009, 인용자 유형별 가중 근거)

### 5.2 ForwardCitationQuality (가중 0.3)

**한 줄 정의**: 본 특허를 인용한 특허들이 얼마나 권위 있는가

| 항목 | 내용 |
|---|---|
| 원시값 | 본 특허를 인용한 특허들의 PageRank 평균 |
| 변환 | `log(1 + 평균 PageRank)` |
| 정규화 | 같은 WIPO 분류 + 등록연도 ±2년 코호트 내 백분위 |
| 출력 | [0, 1] |
| 데이터 | PCG 전역 PageRank 선계산 결과 |

**왜 중요한가**: "권위 있는 특허들이 인용한 특허는 그 자체로 더 가치 있다"는 연쇄 신호. 인용 수(양)만 보는 ForwardCitationCount를 인용 품질(질)로 보완.

**참고 문헌**: Carpenter, Narin & Woolf (1981)

### 5.3 CitationSpan (가중 0.2)

**한 줄 정의**: 인용한 특허들이 얼마나 다양한 분야에 퍼져 있는가

| 항목 | 내용 |
|---|---|
| 원시값 | 본 특허를 인용한 특허들의 IPC 분포 |
| 변환 | Shannon 엔트로피 ÷ log(관측 카테고리 수) |
| 정규화 | 변환만으로 [0, 1] |
| 출력 | [0, 1] |
| 데이터 | PCG 인입 엣지 |

**왜 중요한가**: 여러 분야로 파급된 특허는 범용 기술일 가능성이 높음. 단, 분야 간 인용은 노이즈가 많아 보조 신호.

**참고 문헌**: OECD 2013 generality 지수

## 6. 축 3 — MKT 시장성

```text
MKT(p) = 0.5 × FamilySize
       + 0.4 × IndustryAlignment
       + 0.1 × IsSEP

IndustryAlignment = 0.4 × TAM
                  + 0.3 × CAGR
                  + 0.3 × PatentHiringDemand
```

패밀리 사이즈는 출원인이 직접 비용을 지불한 결정이라 신뢰도가 가장 높아 0.5. 시장 매칭(0.4)이 보조. SEP는 매우 가치 있으나 0.01% 미만에 해당하므로 0.1로 작은 보너스.

### 6.1 FamilySize (가중 0.5)

**한 줄 정의**: 몇 개 나라에 출원했는가

| 항목 | 내용 |
|---|---|
| 원시값 | INPADOC 패밀리 국가 수 |
| 변환 | `log(1 + 국가 수)` |
| 정규화 | 같은 WIPO 분류 + 출원연도 ±2년 코호트 내 백분위 |
| 출력 | [0, 1] |
| 데이터 | KIPRIS Plus 패밀리, EPO OPS |

**왜 중요한가**: 다국가 출원은 출원인이 비용(국가당 수천만 원)을 지불한 자발적 결정이므로 가치 신호가 매우 강하다. LexisNexis PatentSight의 Market Coverage가 이 지표의 GNI 가중 버전.

**참고 문헌**: Putnam (1996), Harhoff, Scherer & Vopel (2003)

### 6.2 IndustryAlignment (가중 0.4)

특허가 매핑된 산업이 얼마나 크고, 얼마나 성장하며, 얼마나 인력 수요가 있는가를 종합. KSIC-IPC 연계표(KIPO 공식)로 IPC를 산업 코드에 매핑.

**TAM (서브 가중 0.4)** — 산업의 시장 규모

| 항목 | 내용 |
|---|---|
| 원시값 | 매핑된 KSIC 산업의 매출액 (KRW) |
| 변환 | `log(1 + 매출액)` |
| 정규화 | 전체 KSIC 산업 코드 풀 내 백분위 |
| 출력 | [0, 1] |
| 데이터 | KIPO KSIC-IPC 연계표 + KOSIS |

**CAGR (서브 가중 0.3)** — 산업의 성장률

| 항목 | 내용 |
|---|---|
| 원시값 | 매핑된 산업의 직전 5년 연평균 성장률 |
| 변환 | `clip(CAGR, −0.2, 0.5)` — 극단값 컷 |
| 정규화 | 전체 KSIC 산업 풀 내 백분위 |
| 출력 | [0, 1] |
| 데이터 | KISDI / KISTEP 산업동향 |

> 시장 규모(stock)만 보면 신생 분야가 불리해지므로 성장률(flow)을 병용. Schmookler(1966) 수요 견인 가설을 측정 가능한 형태로 운영.

**PatentHiringDemand (서브 가중 0.3)** — 분야 인력 수요

| 항목 | 내용 |
|---|---|
| 원시값 | 본 특허 IPC 서브클래스에 매핑된 직무의 최근 12개월 신규 채용 공고 수 |
| 변환 | `log(1 + 채용 공고 수)` |
| 정규화 | WIPO 분류 내 IPC 풀에서 백분위 |
| 출력 | [0, 1] |
| 데이터 | 워크넷, 사람인 등 |

> 산업이 실제로 그 기술 인력을 원하고 있는가를 보는 실시간 수요 신호.

### 6.3 IsSEP (가중 0.1, 이진)

**한 줄 정의**: 표준 필수 특허인가

| 항목 | 내용 |
|---|---|
| 원시값 | ETSI / IEEE / 3GPP DB에서 SEP 선언 여부 |
| 변환·정규화 | 없음 |
| 출력 | {0, 1} |

**왜 중요한가**: SEP는 가치가 매우 크지만 전체 특허의 0.01% 미만. 가중을 크게 두면 99.99%의 특허에서 신호가 죽으므로, 이진 × 0.1 = SEP에만 보너스 부여하는 구조.

## 7. 축 4 — NET 네트워크 중심성

```text
NET(p) = 0.4 × PageRank
       + 0.2 × Betweenness
       + 0.2 × CrossDomainBridgeScore
       + 0.2 × TemporalPersistence
```

PageRank는 가장 강력하고 잘 검증된 그래프 신호로 0.4. 나머지 셋은 보조 신호로 각 0.2.

> 본 축은 §3의 PCG 전역 연산이 전제다. PCG 없이는 PageRank, Betweenness, CrossDomainBridgeScore가 계산되지 않는다.

### 7.1 PageRank (가중 0.4)

**한 줄 정의**: 인용 그래프에서 얼마나 권위 있는 위치인가

| 항목 | 내용 |
|---|---|
| 원시값 | 특허 인용 그래프 PageRank (damping = 0.85) |
| 변환 | `log(1 + PageRank)` |
| 정규화 | 같은 WIPO 분류 + 등록연도 ±2년 코호트 내 백분위 |
| 출력 | [0, 1] |
| 데이터 | PCG 전역 계산 |

**왜 중요한가**: "권위 있는 특허에게 인용받는 특허는 권위가 있다"는 연쇄 정의로 계산되는 글로벌 위상 지표. IMP의 ForwardCitationQuality와 비슷해 보이지만, IMP는 "인용한 측의 PageRank 평균"이고 NET은 "본인의 PageRank"라는 점에서 다르다.

**참고 문헌**: Page, Brin, Motwani & Winograd (1998)

### 7.2 Betweenness (가중 0.2)

**한 줄 정의**: 분야와 분야를 잇는 다리 역할인가

| 항목 | 내용 |
|---|---|
| 원시값 | 특허 인용 그래프 매개 중심성 |
| 변환 | `log(1 + Betweenness)` |
| 정규화 | 같은 WIPO 분류 + 등록연도 ±2년 코호트 내 백분위 |
| 출력 | [0, 1] |
| 데이터 | PCG 전역 계산 (30만 노드 초과 시 sampling 근사) |

**왜 중요한가**: 정보 흐름의 병목에 위치한 특허는 융합·교량 기술인 경우가 많다.

### 7.3 CrossDomainBridgeScore (가중 0.2)

**한 줄 정의**: 서로 다른 기술 영역을 얼마나 연결하는가

```text
CDBS(p) = (서로 다른 IPC 섹션을 잇는 인용 쌍 수) / (전체 인용 쌍 수)
```

| 항목 | 내용 |
|---|---|
| 원시값 | 본 특허의 인용/피인용 IPC 쌍 분포 |
| 변환·정규화 | 비율이므로 그대로 [0, 1] |
| 출력 | [0, 1] |
| 데이터 | PCG 엣지 |

**왜 중요한가**: INV의 IPCDiversity는 본인의 IPC 다양성이지만, CDBS는 본인이 외부로 연결하는 IPC의 다양성이라 의미가 다르다.

### 7.4 TemporalPersistence (가중 0.2)

**한 줄 정의**: 오랫동안 꾸준히 인용되는가

```text
TP(p) = 1 − Gini(연차별 피인용 수)
```

| 항목 | 내용 |
|---|---|
| 원시값 | 등록 후 연차별 피인용 수 |
| 변환 | `1 − Gini` |
| 정규화 | 그대로 [0, 1] |
| 출력 | [0, 1] |
| 결측 처리 | 등록 5년 미만은 결측 → 코호트 평균으로 채움 |

**왜 중요한가**: 같은 누적 100건이라도 "초기 폭발 후 사라진" vs "꾸준히 인용된" 특허는 다른 신호.

## 8. 축 5 — COM 사업화 준비도

```text
COM(p) = 0.35 × HasSpinoff
       + 0.25 × InventorCommercializationIndex (ICI)
       + 0.20 × HasFollowOnFunding
       + 0.20 × ProjectFundingScale

ICI = 0.3 × PriorPatentCount
    + 0.3 × StartupLinkage
    + 0.2 × FundedProjectsCount
    + 0.2 × CollaborationCentrality
```

스핀오프는 사업화 의도가 실제 실행된 가장 강한 신호이므로 0.35. 발명자 신호(ICI) 0.25, 후속 투자 0.20, 과제 예산 규모 0.20.

### 8.1 HasSpinoff (가중 0.35, 이진)

**한 줄 정의**: 이 특허로 스핀오프 기업이 나왔는가

| 항목 | 내용 |
|---|---|
| 원시값 | 본 특허가 기술이전을 통해 스핀오프 기업과 연결되는지 |
| 변환·정규화 | 없음 |
| 출력 | {0, 1} |
| 데이터 | NTIS 기술이전, DART, KOSDAQ/KONEX 신규상장 매칭, 보도자료 |

**왜 중요한가**: 1인 경우는 드물지만 발생 시 강력한 신호. 가중치 0.35는 1이 될 때만 +35점을 부여하는 보너스 구조.

### 8.2 InventorCommercializationIndex (가중 0.25)

발명자의 사업화 친화도를 4개 서브 변수로 측정.

**PriorPatentCount (서브 0.3)** — 발명자의 과거 등록 특허 수

| 항목 | 내용 |
|---|---|
| 원시값 | 본 특허 출원 시점까지 발명자(들)의 누적 등록 특허 수 |
| 변환 | `log(1 + N)` |
| 정규화 | 전체 발명자 풀 내 백분위 |
| 출력 | [0, 1] |
| 데이터 | KIPRIS Plus |

**StartupLinkage (서브 0.3, 이진)** — 발명자가 회사 운영 중인가

| 항목 | 내용 |
|---|---|
| 원시값 | 발명자가 기업의 창업자/공동창업자/임원으로 등재 |
| 변환·정규화 | 없음 |
| 출력 | {0, 1} |
| 데이터 | DART 임원현황, 더브이씨, 보도자료 |

**FundedProjectsCount (서브 0.2)** — 발명자의 정부 R&D 과제 경험

| 항목 | 내용 |
|---|---|
| 원시값 | 발명자가 책임/공동연구원으로 참여한 NTIS 과제 건수 |
| 변환 | `log(1 + N)` |
| 정규화 | 전체 발명자 풀 내 백분위 |
| 출력 | [0, 1] |
| 데이터 | NTIS |

**CollaborationCentrality (서브 0.2)** — 발명자 협력 네트워크에서의 중심성

| 항목 | 내용 |
|---|---|
| 원시값 | 발명자의 ICG 내 가중 연결중심성 |
| 변환 | `log(1 + degree)` |
| 정규화 | 전체 발명자 풀 내 백분위 |
| 출력 | [0, 1] |
| 데이터 | §3.2 ICG |

### 8.3 HasFollowOnFunding (가중 0.20, 이진)

**한 줄 정의**: 출원 후 후속 투자가 들어왔는가

| 항목 | 내용 |
|---|---|
| 원시값 | 본 특허 출원 후 24개월 내 관련 기업의 Seed 이상 투자 유치 여부 |
| 변환·정규화 | 없음 |
| 출력 | {0, 1} |
| 데이터 | 더브이씨, Crunchbase |
| 결측 처리 | 출원 후 24개월 미경과 시 결측 (모집단 평균 imputation) |

### 8.4 ProjectFundingScale (가중 0.20)

**한 줄 정의**: 이 특허와 연결된 R&D 과제 예산 규모

| 항목 | 내용 |
|---|---|
| 원시값 | 본 특허와 연계된 NTIS 과제 총 예산 (KRW) |
| 변환 | `log(1 + 예산)` |
| 정규화 | 전체 NTIS 과제 풀 내 백분위 |
| 출력 | [0, 1] |
| 데이터 | NTIS 과제 + 사사 표기 기준 매핑 |

## 9. 최종 점수 산출

### 9.1 1단계 — 5축 가중합

```text
PatentScore_raw(p) = 0.25 × INV(p)
                   + 0.20 × IMP(p)
                   + 0.20 × MKT(p)
                   + 0.20 × NET(p)
                   + 0.15 × COM(p)
```

각 축 점수는 [0, 1] 범위이므로 가중합도 자연스럽게 [0, 1] 범위.

### 9.2 2단계 — WIPO 분류 내 스테나인 변환

**왜 한 번 더 변환하는가**

§2에서 각 변수를 코호트 정규화했지만, 5축 가중합 자체는 절대 점수다. 분야 간 비교 가능성을 확보하고, **0.62와 0.63의 미세 차이를 과장 해석하지 않기 위해** 한 번 더 등급화한다.

**스테나인이란**

정규화된 9등급 점수 체계. 미국 공군이 2차 대전기에 개발한 표준점수 시스템으로, 정해진 비율에 따라 1~9 점수를 부여한다.

| 등급 | 비율 | 누적 |
|---:|---:|---:|
| 9 | 4% | 100% |
| 8 | 7% | 96% |
| 7 | 12% | 89% |
| 6 | 17% | 77% |
| 5 | 20% | 60% |
| 4 | 17% | 40% |
| 3 | 12% | 23% |
| 2 | 7% | 11% |
| 1 | 4% | 4% |

**구현**

원점수의 분포가 정규성을 보장하지 않으므로, **순위 비율로 직접 스테나인을 부여**한다.

```text
def to_stanine(score, cohort_scores):
    pct = percentile_rank(score, cohort_scores)   # 0 ~ 1
    cumulative = [0.04, 0.11, 0.23, 0.40, 0.60, 0.77, 0.89, 0.96, 1.00]
    for s, c in enumerate(cumulative, start=1):
        if pct <= c:
            return s
    return 9
```

- 코호트: WIPO 4분류 (정밀이 필요하면 IPC 섹션 단위)

### 9.3 3단계 — AAA ~ C 등급 매핑

| 스테나인 | 등급 | 비율 |
|---:|:---:|---:|
| 9 | **AAA** | 4% |
| 8 | **AA** | 7% |
| 7 | **A** | 12% |
| 6 | **BBB** | 17% |
| 5 | **BB** | 20% |
| 4 | **B** | 17% |
| 3 | **CCC** | 12% |
| 2 | **CC** | 7% |
| 1 | **C** | 4% |

**출력 예시**

```text
Patent ID:        KR10-2023-0123456
WIPO Class:       전기·전자·IT
축별 점수:
  INV: 0.78    IMP: 0.65    MKT: 0.55
  NET: 0.71    COM: 0.43
PatentScore_raw:  0.654
코호트 백분위:    89.7%
Stanine:          8  →  Grade: AA
```

## 10. 가중치 검증과 보정

현재 가중치는 학술 문헌(Tong & Frame, Harhoff, Trajtenberg, OECD 등)에 근거한 **초기 설정**이다. 운영하면서 다음 절차로 보정한다.

### 10.1 변수 자격 조건 — 정량성·객관성·완전성

모든 변수는 세 가지 조건을 충족해야 한다.

| 조건 | 의미 | 위반 시 |
|---|---|---|
| **정량성** | 개인 주관과 독립. 누구나 동일한 답이 나옴 (예: 독립청구항이 5개인가? Yes) | 변수 제거 |
| **객관성** | 수학적으로 측정 가능. 수치(0, 1, 2, …) 또는 논리(0/1) | 변수 제거 |
| **완전성** | 모든 평가 대상에 데이터가 존재 (예: 매출 정보는 기업 특허에만 있으므로 미충족) | 결측 처리 규칙 명시 |

각 변수의 충족 여부는 §11 점검표 참조.

### 10.2 델파이 전문가 검증 (3라운드)

1. **1라운드**: 패널 30명에게 무기명으로 각 변수의 중요도와 5축 가중치 적정성 평가
2. **2라운드**: 1라운드의 중앙값과 사분범위를 피드백 → 의견 수렴
3. **3라운드**: 합의 도출

### 10.3 CVR (Content Validity Ratio) ≥ 0.33

```text
CVR = (n_essential − N/2) / (N/2)
```

- `n_essential`: "필수"라고 답한 패널 수
- `N`: 전체 패널 수
- 패널 30명 기준 임계값 0.33
- 임계 미달 변수는 가중치 0으로 전환 검토

### 10.4 Cronbach α ≥ 0.7

각 축 내부 변수들이 같은 개념을 측정하는지 확인. α가 0.7 미만이면 해당 축의 변수 재정의 또는 가중치 재배분.

### 10.5 실거래 데이터 기반 보정 (장기)

실제 사업화 결과(라이선스 매출, M&A 거래가, 소송 승률 등)가 100건 이상 누적되면 다중회귀로 가중치를 재추정한다.

```text
y₁ = β₀¹ + β₁¹·INV + β₂¹·IMP + β₃¹·MKT + β₄¹·NET + β₅¹·COM + ε   (라이선스 매출)
y₂ = β₀² + β₁²·INV + β₂²·IMP + β₃²·MKT + β₄²·NET + β₅²·COM + ε   (거래가)
y₃ = β₀³ + β₁³·INV + β₂³·IMP + β₃³·MKT + β₄³·NET + β₅³·COM + ε   (소송 승률)
```

사용 시나리오별로 회귀계수를 가중 결합하여 최종 점수에 반영:

```text
Y_final = (α₁·β₁¹ + α₂·β₁² + α₃·β₁³) · INV + …
```

- 금융기관은 거래가(y₂)에 가중
- 라이선싱 회사는 매출(y₁)에 가중
- 분쟁부서는 승률(y₃)에 가중
