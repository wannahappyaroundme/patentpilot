# PatentRank

## 5축 스코어 공식 — 가중치 근거 (최종판)

# 1. 축간 가중치 근거

| 축 | 가중치 | 역할 |
|---|---:|---|
| 1. INV — 혁신성 | 0.25 | 최대 가중치 |
| 2. IMP — 영향력 | 0.20 |  |
| 3. MKT — 시장성 | 0.20 |  |
| 4. NET — 네트워크 중심성 | 0.20 |  |
| 5. COM — 사업화 준비도 | 0.15 | 최소 가중 |

## 축별 비중 사유

- **INV (0.25):** NET 축이 엄격해지면서 INV의 상대적 중요도 상승. ClaimBreadth 와 IPCDiversity는 가장 빨리 안정되는 신호.
- **IMP (0.20):** NET의 PageRank / HITS 와 중복 제거 후 합리적 비중 축소.
- **MKT (0.20):** 유지.
- **NET (0.20):** 유지.
- **COM (0.15):** binary 문제를 완화 후 점수 상향.

# 2. 축 1 — INV 혁신성

```text
INV(p) = 0.4 × ClaimBreadth(p)
       + 0.3 × IPCDiversity(p)
       + 0.3 × BackwardCitationDiversity(p)
```

## ClaimBreadth = 0.4 (최대)

독립청구항 수는 특허의 권리 범위(scope)를 직접 반영합니다. Tong & Frame(1994)는 청구항 가중 특허수가 단순 특허수보다 R&D 지출과 더 강하게 상관함을 보였고, Lanjouw & Schankerman(2001)은 청구항 수가 소송 확률과 유의한 양의 상관을 가짐. 즉 혁신성 지표의 가장 직접적인 지표.

## IPCDiversity = 0.3

서로 다른 IPC 서브클래스 수는 기술의 폭(breadth)을 나타내며, OECD 2013에서 originality(Trajtenberg-Henderson-Jaffe 1997) 지표의 한 형태로 사용. 다양한 분류에 걸치는 특허는 융합기술 / 플랫폼 기술인 경향이 있어 혁신성 지표로 활용.

## BackwardCitationDiversity = 0.3

인용한 선행 특허의 IPC 분포 엔트로피는 OECD 2013의 originality 지표 원형과 가장 가까움. 다양한 분야에서 지식을 끌어왔다는 것 = 새로운 결합 = 혁신적이라는 슘페터적 가정. Carpenter, Narin & Woolf(1981)가 시초.

## 0.4 vs 0.3 vs 0.3 차등 이유

Claim 수는 출원 시점에 즉시 확정되는 강한 신호이지만, IPC / backward diversity는 분류 체계와 심사관 재량에 의존하는 약한 신호. 따라서 가장 견고한 ClaimBreadth에 더 큰 가중치 채택.

# 3. 축 2 — IMP 영향력

```text
IMP(p) = 0.5 × ForwardCitationCount_Norm(p, age)
       + 0.3 × ForwardCitationQuality(p)
       + 0.2 × CitationSpan(p)
```

## ForwardCitationCount_Norm = 0.5 (최대)

5축 합산 차원에서 IMP가 0.30, 그 안에서 0.5 → 전체 PatentRank에서 단일 변수로 가장 큰 가중. 이는 학술 합의(Trajtenberg 1990, Harhoff et al. 1999, Hall et al. 2005)와 정합. 단, 오래된 특허만 유리해짐을 보완하기 위해 age 정규화(IPC별 평균 인용 곡선)가 필수.

## ForwardCitationQuality = 0.3

“고품질 특허들이 인용한 특허는 더 가치 있다”는 PageRank-style 재귀 신호. Page & Brin(1998) 원리를 NET 축에서도 사용하지만, IMP 축에서는 인용한 측의 PageRank 평균만 사용해 방향성 신호를 분리. Carpenter, Narin & Woolf(1981)의 “high-impact patents are cited by other high-impact patents” 가설 운영화.

## CitationSpan = 0.2

인용 특허들의 IPC 다양성. OECD 2013의 generality 지수(forward citation 측). 타분야로 파급된 특허는 범용 기술(general purpose technology)일 가능성이 높음. 단, 분야 간 인용은 노이즈가 많아 가중치 낮게 채택.

## 0.5 vs 0.3 vs 0.2 차등 이유

Hegde & Sampat(2009): “examiner citations are stronger predictors than applicant citations” → 2.5절의 examiner = 1.5, applicant = 0.7 가중치는 이 논문에 직접 근거. 단순 인용 수가 가장 강한 신호이므로 0.5는 합리적임. Quality(0.3)는 양보다 질을 보완. Span(0.2)은 보조 신호.

# 4. 축 3 — MKT 시장성

```text
MKT(p) = 0.5 × FamilySize_Norm(p)
       + 0.4 × IndustryAlignment(p)
       + 0.1 × IsSEP(p) {0,1}

IndustryAlignment = 0.4 × TAM
                  + 0.3 × CAGR
                  + 0.3 × PatentHiringDemand
```

## FamilySize_Norm = 0.5 (최대)

패밀리 사이즈는 출원인이 직접 비용을 지불한 의사결정이므로 자기선택편의(self-selection bias)가 적음. Putnam(1996) 박사논문이 시초, Harhoff, Scherer & Vopel(2003)이 Research Policy에서 “family size > forward citations as a value indicator” 입증. LexisNexis PatentSight의 Market Coverage가 이 지표의 GNI 가중 버전.

특허가 매핑된 산업의 시장 규모. KSIC-IPC 연계표(KIPO 발간)와 산업별 GDP 기여도를 결합. 시장 규모와 특허 가치의 정합성을 강조한 Schmookler(1966) demand-pull 이론의 운영화.

## IndustryAlignment(p) = 0.4

- **TAM (Total Addressable Market) = 0.4** : 특허가 매핑된 산업의 시장 규모. KISDI, KISTEP, KOSIS 산업별 매출 데이터. 한국 KSIC 코드 단위 매출액(통계청)을 시장 규모 대용으로 활용 가능.
- **CAGR (Compound Annual Growth Rate) = 0.3** : 해당 산업의 5년 평균 성장률. 시장 규모(stock)만으로는 신생 deep-tech 분야가 불리해지는 문제 보완. KISDI / KISTEP 산업 동향 보고서 기반.
- **PatentHiringDemand = 0.3** : 해당 IPC 분야 채용 공고 수(워크넷, LinkedIn). 외부 리뷰의 핵심 권고 — “산업이 실제 기술을 원하고 있는가”를 측정.

## IsSEP = 0.1 {0,1}

표준 필수 특허는 매우 가치 있으나 극히 소수(ETSI IPR DB 기준 전체 특허의 0.01% 미만). 가중치를 높이면 SEP가 아닌 99.99%의 특허에서 신호가 죽음. 이진 × 0.1 가중으로 SEP에만 점수를 부여하는 효과.

# 5. 축 4 — NET 네트워크 중심성

```text
NET(p) = 0.4 × PageRank
       + 0.2 × Betweenness
       + 0.2 × CrossDomainBridgeScore
       + 0.2 × TemporalPersistence
```

## PageRank = 0.4 (다수)

Page, Brin, Motwani & Winograd(1998)의 PageRank 원논문은 damping = 0.85를 표준값으로 권고, PatentRank도 이를 채용(2.5절). 특허 인용 그래프에서 PageRank는 재귀적 권위(recursive authority)를 측정하며, IMP의 ForwardCitationQuality와 상관이 있지만 글로벌 위상학적 위치를 본다는 점에서 다름. 따라서 가중치 0.4.

## Betweenness = 0.2

특허 그래프에서 한 노드의 매개 중심성.

## CrossDomainBridgeScore = 0.2

서로 다른 IPC cluster 연결하는 특허 변수.

## TemporalPersistence = 0.2

특허의 인용 활동이 시간에 걸쳐 지속되는 속도. 숨어 있는 특허 식별의 보조 신호로 활용.

- 등록 후 N년 단위 인용 수의 분산 또는 시계열 안정성 측정.
- 단순 누적 인용수와 다른 정보 - 같은 누적 인용 100건이라도 “초기 3년 100건 후 0건” vs “20년에 걸쳐 평균 5건 / 년”이 다름.

# 6. 축 5 — COM 사업화 준비도

```text
COM(p) = 0.35 × HasSpinoff(p) {0,1}
       + 0.25 × InventorCommercializationIndex(p)
       + 0.20 × HasFollowOnFunding(p) {0,1}
       + 0.20 × ProjectFundingScale(p)

InventorCommercializationIndex = 0.25
    = 0.3 × Prior patents
    + 0.3 × startup linkage
    + 0.2 × funded projects
    + 0.2 × collaboration centrality
```

## HasSpinoff = 0.35 (최대, 이진)

대학발 창업기업이 발명에서 비롯되었다는 것은 사업화 의도가 실행된 가장 강한 신호. 단 이진값이라 0인 특허가 다수. 0.35 가중치는 1이 될 때만 +35점 부여하는 강한 보너스 구조.

## InventorCommercializationIndex = 0.25

- **PriorPatentCount = 0.3** : 발명자 과거 등록 특허 건수. KIPRIS Plus 에서 직접 측정 가능.
- **StartupLinkage = 0.3 {0,1}** : 발명자가 기업의 창업자 / 공동창업자 / 임원인지 여부. DART / 더브이씨 / 스타트업 명단으로 측정, 보도자료 스크랩 매칭.
- **FundedProjectsCount = 0.2** : NTIS 에서 발명자가 책임자 또는 공동연구원으로 참여한 정부 R&D 과제 건수.
- **CollaborationCentrality = 0.2** : 발명자의 공동발명 네트워크에서 중심성. KIPRIS 공동발명자 네트워크로 측정.
